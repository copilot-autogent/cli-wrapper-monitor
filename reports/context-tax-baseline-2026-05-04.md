## Interpretation

### 6.5% context consumption at session start

With claude-opus-4.6's 200k context window, ~12,956 tokens are consumed before the user sends a single message. This is the **floor** — the minimum cost of any session.

For the model's effective working memory:
- 187,044 tokens remain for user conversation + tool calls + responses
- A typical 1-hour autogent session with 20 turns × ~2k tokens/turn consumes ~40k additional tokens
- **Normal sessions never approach context limits** — the overhead is within budget

### However: this is not static

The system prompt grows as:
- Bootstrap files are updated (PLAYBOOK additions, CONTEXT notes)
- New tools are added (each adds ~41 tokens)
- Cross-instruction file content expands

**At current growth rate** (autogent averages ~2-3 PLAYBOOK updates/week, ~500 chars each), the system prompt could grow 10% in 6-8 weeks. The 10% regression threshold triggers a warning.

### Regression trigger conditions

| Change | Token impact | Flag? |
|--------|-------------|-------|
| Add one tool | +41 tokens (+0.3%) | ⚪ No |
| Add 10 tools | +410 tokens (+3.2%) | ⚪ No |
| Major PLAYBOOK addition (~2k chars) | +500 tokens (+3.9%) | ⚪ No |
| 3× PLAYBOOK additions | +1,500 tokens (+11.6%) | 🔴 Yes |
| Bootstrap file restructure | varies | 🟡 Monitor |

The most likely regression source is **incremental PLAYBOOK growth** accumulating over months without any single addition crossing the threshold.

## What This Baseline Enables

1. **Drift detection**: Monthly snapshots will show whether the system prompt is growing, stable, or shrinking after optimizations.
2. **Tool cost accounting**: Each new tool has a measurable cost. Before adding a tool, the baseline makes that cost visible.
3. **Before/after verification**: Any "system prompt optimization" effort can be quantified. Did the cleanup actually save tokens? By how much?
4. **Cross-version comparison**: When the CLI upgrades, does the wrapper inject more overhead? This baseline answers that.
